# CC-Visual Roadmap

> 一眼看出哪个 AI agent 需要你介入——然后一键跳过去。

## 设计原则

产品的核心循环只有三步：**看到谁在等你 → 点击 → 跳转到那个窗口**。所有设计决策都服务于让这三步更快、更直觉。不服务于这个循环的功能，要么降级为二级入口，要么删掉。

### 自检清单

每次加功能前问自己：

1. 这个功能有没有让用户更快完成"看到→点击→跳转"？
2. 第一次打开的用户 3 秒内能知道该干什么吗？
3. 信息层级是在帮用户聚焦，还是在分散注意力？

三个答案不全是"是"，这个功能就不该做——或者不该以当前形式做。

### v1.0 问题诊断

功能做完了，但产品体验出了问题：

| 问题 | 现状 | 影响 |
|------|------|------|
| 信息层级混乱 | 8 个 z-index 层、9 个面板区域 | 用户不知道该看哪里 |
| 像素字体滥用 | 5-8px Press Start 2P 渲染数据 | 数据难以阅读 |
| 中英文混搭 | 6 处中文混在英文 UI 里 | 定位模糊 |
| 地图寻宝 | 角色随机散落在 50×36 地图上 | 找 waiting 角色比看列表还慢 |
| 功能平铺 | 统计/诊断/时间线/设置同级展示 | 核心操作被淹没 |
| 无首次引导 | 打开即全量展示 | 新用户 3 秒内无法理解该做什么 |

**结论：不需要 S10。需要的是删掉一半功能的勇气。**

## R1: 状态驱动布局

**目标**：打开应用，需要你介入的 agent 自动站到最显眼的位置。不用在地图上找。

### 设计理念

当前角色位置由随机种子决定，与状态无关。改为：**状态决定位置区域**。

```
地图布局：

  ┌─────────────────────────────────┐
  │         边缘区域 (Idle)          │
  │   ┌─────────────────────────┐   │
  │   │   中间区域 (Running)     │   │
  │   │   ┌─────────────────┐   │   │
  │   │   │  舞台中央        │   │   │
  │   │   │  (Waiting)      │   │   │
  │   │   │  最大、最亮     │   │   │
  │   │   └─────────────────┘   │   │
  │   └─────────────────────────┘   │
  └─────────────────────────────────┘
```

- **Waiting**：聚集在地图中央「舞台区」，面朝镜头，头顶感叹号。等待越久越靠前。
- **Running**：在中间环形区域正常行走（保留活力感）。
- **Idle**：半透明，退到地图边缘，不抢注意力。
- 状态变化时角色平滑走向新区域（不瞬移）。
- 相机有 Waiting 角色时自动对焦到舞台区。

### 改动范围

`src/client/character.ts`（位置分配逻辑）、`src/client/game.ts`（相机对焦）、`src/client/world.ts`（可选：舞台区地砖视觉区分）

### 验收标准

- [ ] Waiting 角色自动聚集在地图中央区域
- [ ] 多个 Waiting 角色按等待时长排序，最久的在最前面
- [ ] Running 角色在中间区域活动，不进入舞台中央
- [ ] Idle 角色退到边缘，视觉上不抢眼（半透明或缩小）
- [ ] 状态变化时角色平滑移动到新区域，不瞬移
- [ ] 相机有 Waiting 角色时自动居中到舞台区
- [ ] 无 Waiting 角色时相机居中到所有活跃角色的中心
- [ ] 角色外观仍由 sessionId 种子决定（不破坏确定性生成）

## R2: 信息层级瘦身

**目标**：从 9 个信息层降到 3 层。用户视线有且只有一个焦点。

### 三层结构

```
第一层（永远可见）：谁在等你
  - HUD 栏只保留：Waiting 计数 + 静音按钮 + 诊断入口
  - 侧边栏：精简会话列表（见下方）
  - Canvas 中央：Waiting 角色（来自 R1）

第二层（点击展开）：这个会话怎么了
  - 右侧面板精简为：当前工具 + 最近回复 + 错误（如有）+ 跳转按钮
  - 最多 4 个区域

第三层（主动寻找）：高级功能
  - 统计面板、诊断面板、时间线 → 合并为一个「详情」入口
  - 设置面板 → 保留但入口缩小
```

### 侧边栏精简

当前每个 session 有 6 个区域、48+ 数据点。精简为：

```
┌──────────────────────────────┐
│ ● my-project     WAITING  ▶ │
│   waited 5 min               │
└──────────────────────────────┘
│ ● api-server     RUNNING    │
│   Edit · 42 calls · 3m ago  │
└──────────────────────────────┘
```

- 第一行：状态点 + 项目名 + 状态标签（+ Waiting 跳转按钮）
- 第二行：Waiting 显示等待时长；Running/Idle 显示当前工具 + 调用次数 + 时间
- 去掉一级展示：Session ID、来源 badge、已看按钮、alert 图标（降级到面板中查看）

### 改动范围

`src/client/sidebar.ts`（精简渲染）、`src/client/panel.ts`（面板降维）、`src/client/game.ts`（HUD 精简）、`public/css/style.css`

### 验收标准

- [ ] 侧边栏每个 session 最多两行信息
- [ ] Waiting session 有醒目的跳转按钮，点击直接触发窗口聚焦
- [ ] HUD 栏只有 3 个元素：Waiting 计数、静音、诊断入口
- [ ] 右侧面板最多 4 个区域（当前工具 + 最近回复 + 错误 + 操作按钮）
- [ ] 统计/诊断/时间线合并为单一「详情」二级入口
- [ ] 被降级的信息在面板中仍可查看（不是删除，是移到第二层/第三层）
- [ ] 筛选和搜索功能保留，但视觉权重降低（不占用第一行空间）

## R3: 字体分层

**目标**：像素字体只用在标题和氛围元素上，数据用可读字体。

### 字体规则

| 场景 | 字体 | 字号 |
|------|------|------|
| 应用标题、Logo | Press Start 2P | 11-14px |
| 角色名牌、阶段 tag | Press Start 2P | 6-8px |
| HUD 状态标签（WAITING 计数等） | Press Start 2P | 8-9px |
| 侧边栏项目名、状态文本 | `--mono`（SF Mono / Consolas） | 11-12px |
| 面板数据（工具名、时间、错误） | `--mono` | 11-12px |
| 面板描述文本 | system-ui, sans-serif | 12-13px |

关键变化：
- 侧边栏从 5-7px 像素字体 → 11-12px 等宽字体
- 面板数据从 7-8px 像素字体 → 11-12px 等宽字体
- 像素字体只保留在 Canvas 上的角色标签和 HUD 装饰性元素
- **底线：没有任何 UI 文本小于 10px**

### 改动范围

`public/css/style.css`（字体变量和应用规则）

### 验收标准

- [ ] 侧边栏文本使用等宽字体，最小 11px
- [ ] 面板数据使用等宽字体，最小 11px
- [ ] Canvas 上的角色名牌/阶段 tag 保持像素字体
- [ ] HUD 标题和状态标签保持像素字体
- [ ] 整体视觉仍然有像素风氛围（靠 Canvas + 颜色 + HUD 标题维持）
- [ ] 没有任何 UI 文本小于 10px

## R4: 语言统一 + 首次体验

**目标**：统一为英文 UI。首次打开 3 秒内知道该做什么。

### 语言统一

| 当前（中文） | 改为（英文） |
|-------------|-------------|
| 已看 | Seen |
| 等你确认 | Needs you |
| 搜索中 | Searching |
| 编辑中 | Editing |
| 执行命令 | Running cmd |
| 工作中 | Working |

### 首次体验引导

不做教程弹窗——用**空状态设计**自然引导：

| 场景 | 用户看到什么 |
|------|-------------|
| 无会话 | 一个像素角色站在舞台中央，头顶气泡 "Start a Claude Code or Codex session — I'll appear here" |
| 有会话，无 Waiting | 角色们正常活动，HUD 显示 "All clear — 0 waiting" |
| 有 Waiting | 舞台区角色头顶感叹号，HUD 闪烁 "⚠ 2 waiting"，侧边栏 Waiting 项脉冲动画 |

三种状态自然告诉用户这个产品是干什么的。

### 改动范围

`src/client/character.ts`（阶段 tag 英文化）、`src/client/sidebar.ts`（按钮文本）、`src/client/game.ts`（空状态渲染）、`public/css/style.css`

### 验收标准

- [ ] 所有 UI 文本统一为英文，无中英混搭
- [ ] 无会话时显示引导空状态（角色 + 气泡提示）
- [ ] 有 Waiting 时 HUD 有明确视觉提醒（闪烁/脉冲）
- [ ] 侧边栏 Waiting 项有脉冲动画吸引注意力
- [ ] 不使用弹窗或模态对话框做引导

## R5: 一键跳转强化

**目标**：从"看到 Waiting"到"跳到那个终端窗口"不超过 1 次点击。

### 设计理念

当前跳转路径：看到角色 → 点击角色 → 等面板打开 → 找到跳转按钮 → 点击。**4 步。**

目标路径：

| 路径 | 操作 | 结果 |
|------|------|------|
| A（Canvas） | 单击 Waiting 角色 | 直接跳转窗口 |
| B（侧边栏） | 点击 ▶ 按钮 | 直接跳转窗口 |
| C（面板） | 面板顶部跳转按钮 | 直接跳转窗口 |

面板改为双击角色 或 侧边栏点击项目名 打开。**单击 Waiting 角色 = 立即行动，不是打开面板。**

### 改动范围

`src/client/interaction.ts`（点击行为重定义）、`src/client/sidebar.ts`（跳转按钮）、`src/client/panel.ts`（顶部跳转按钮）

### 验收标准

- [ ] 单击 Waiting 角色触发窗口聚焦（不打开面板）
- [ ] 单击 Running/Idle 角色仍打开面板（只有 Waiting 改行为）
- [ ] 双击任意角色打开详情面板
- [ ] 侧边栏 Waiting 项有 ▶ 按钮，点击触发窗口聚焦
- [ ] 面板顶部有常驻跳转按钮
- [ ] 跳转后 toast 确认 "Focused: {project-name}"
- [ ] 窗口聚焦失败时 toast 显示错误原因

## 实施原则

1. **删减优先于新增** — R 系列的核心是减法。能删的删，能降级的降级，能合并的合并
2. **核心循环优先** — 所有改动先问：这让"看到→点击→跳转"更快了吗？
3. **每个迭代独立可交付** — 完成一个迭代就是一个可用版本
4. **像素风在 Canvas，可读性在 UI** — Canvas 保持像素美学，侧边栏和面板追求数据可读性
5. **语言一致** — 所有 UI 文本统一英文，不混搭
6. **最小改动** — 优先改现有文件，不无必要不新建文件
7. **先验后做** — 每个迭代先写验收 checklist，再动手实现
8. **双源一致** — 涉及数据展示的功能须同时验证 Claude Code 和 Codex 两种数据源
9. **渐进增强** — 每个功能都有降级方案，不破坏现有体验
